Done‑for‑you Setup

Why n8n for Automation?

n8n is an open‑source workflow automation platform that lets you connect apps, APIs, and AI models together without writing a full backend service for every integration. It sits in the sweet spot between no‑code tools like Zapier and full custom engineering.

  • Visual workflows – drag‑and‑drop nodes instead of wiring boilerplate code.
  • Open‑source & self‑hostable – keep data where you need it, integrate with anything.
  • Perfect for GenAI – orchestrate prompts, context, and tools around LLMs.

Good mental model: n8n is a programmable pipeline. Data flows from one node to the next, and each node transforms or routes it.

Installing n8n (Cloud & Self‑Hosted)

You can start with n8n.cloud in minutes, or self‑host using Docker if you want full control.

Option 1: n8n.cloud (fastest)

  1. Create an account on n8n.cloud.
  2. Spin up a workspace (the UI will guide you).
  3. Open the editor – you’re ready to build workflows.

Option 2: Self‑host with Docker

If you want data residency or deeper customisation, Docker is the easiest path.


docker run -it \
  --name n8n \
  -p 5678:5678 \
  -e N8N_BASIC_AUTH_ACTIVE=true \
  -e N8N_BASIC_AUTH_USER=admin \
  -e N8N_BASIC_AUTH_PASSWORD=super-secret-password \
  n8nio/n8n

Then open http://localhost:5678 in your browser to access the editor.

Building Your First Workflow in 10 Minutes

Let’s build a simple “new lead → Slack notification” workflow. This pattern shows up in almost every business (lead forms, demo requests, new signups).

  1. Create a new workflow and add a Webhook node.
  2. Configure the webhook as POST and copy the generated URL.
  3. Connect your form or tool (e.g. Webflow, Typeform) to send data to that URL.
  4. Add a Slack node and map the fields you care about (name, email, message).
  5. Click Execute workflow and send a test payload.

{
  "name": "Jane Doe",
  "email": "jane@example.com",
  "company": "Acme Solar",
  "message": "We'd like to automate lead routing from our website."
}

In your Slack node, you can reference these values with n8n’s expression editor, e.g. {{$json["name"]}} or {{$json["email"]}}.

Adding GenAI to Your n8n Workflows

Once you’re comfortable moving data around, the next step is to add GenAI for summarisation, classification, and decision‑making.

Example: Auto‑qualify inbound leads with OpenAI

Pipeline:

  • Webhook receives raw form data.
  • Function node normalises fields.
  • OpenAI node scores the lead based on your criteria.
  • Router node sends “hot” leads straight to sales, others to nurture.

// Example prompt for OpenAI node
const lead = $json;

return {
  prompt: `
You are a sales assistant. Score this inbound lead from 1-5 where 5 is a perfect fit.

Company: ${lead.company}
Role: ${lead.role}
Message: ${lead.message}

Return JSON:
{
  "score": <number 1-5>,
  "summary": "<one sentence summary>",
  "reason": "<why you gave this score>"
}
`
};

Beginner Best Practices

Start Small

Automate one well‑defined workflow first (e.g. lead alerts) before touching your entire stack.

Use Test Data

Always run with realistic but safe test payloads before connecting live systems.

Document Workflows

Add descriptions to nodes so future you (or your team) understand what’s happening.

Think About Failure

What happens if an API is down? Build basic error handling and fallbacks early.

Next Steps: From Toy Workflows to Real Systems

Once you’ve shipped 2–3 simple automations, you’ll start to see patterns: where data lives, how teams collaborate, and where GenAI can make a real difference.

  • Map your “lead to revenue” or “ticket to resolution” journeys.
  • Identify all manual copy‑paste or “check this system, then that system” steps.
  • Group them into workflows and prioritise by effort vs. impact.

We’ve helped teams go from “we just installed n8n” to “40+ hours automated per week” in under 90 days. The compounding effect is huge once you have the first few workflows in place.
